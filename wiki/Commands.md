# Commands Reference

The bot supports a rich set of commands designed to give you complete control over the IDE without needing to see the screen. Many commands have short aliases for rapid typing.

## 💬 Chat Controls

- **`/new`**: Opens a fresh chat thread in the current workspace.
- **`/agents` (or `/a`)**: Lists all recent chat threads across all active workspaces. Clicking a generated link (e.g., `/a1`) switches the IDE to that thread.
- **`/artifacts` (or `/af`)**: Lists all files (images, documents, code) generated by the agent in the current thread. Clicking a link (e.g., `/af1`) will send that file directly to your Telegram chat.
- **`/model` (or `/m`)**: Opens a menu to select which underlying LLM the agent should use for its next turn.
- **`/autoaccept` (or `/aa`)**: Toggles the Auto-Accept state machine on or off.

## 👁️ Status & Info

- **`/latest` (or `/l`)**: Reprints the very last message sent by the agent. Useful if a long response timed out or if you lost context.
- **`/screenshot` (or `/ss`)**: Takes a full-screen image capture of the headless IDE and sends it to you. Great for debugging UI issues.
- **`/status` (or `/s`)**: Displays the health of the system, including PM2 uptime, CDP connection status, and the exact workspace/thread currently active.

## 🛠️ IDE Controls

- **`/start_ide`**: Bootstraps and launches the headless Chrome browser with the Antigravity UI.
- **`/close`**: A hard kill-switch. Terminates the Chrome process and cleans up any orphaned lockfiles.
- **`/workspace`**: Switches the active project folder.
- **`/window`**: If multiple Chrome windows are open, allows you to select which one the bot should control.
- **`/lang`**: Switches the bot's language localization (e.g., English, Turkish).
- **`/cmd <command>`**: Executes a raw terminal command on the host machine.
- **`/help` (or `/h`)**: Displays the quick-reference menu in Telegram.
